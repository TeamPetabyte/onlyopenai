# คู่มือ Trainer — PipekAI

*วิธีเทรน AI ด้วยการทดสอบและตัดสินคำตอบ — อ่าน 10 นาที สำหรับ senior SAP/ABAP developer*
*(English version: [trainer-guide.md](trainer-guide.md))*

> **ทำไมคำตัดสินของคุณถึงสำคัญ:** ทุกคำตอบที่คุณให้คะแนนจะกลายเป็นส่วนหนึ่งของ
> **golden dataset** — AI ไม่ได้ฉลาดขึ้นเองโดยอัตโนมัติ คำตัดสินและเฉลยของคุณ
> คือสิ่งที่เราใช้จูน prompt วัดความแม่นยำ และ (ในอนาคต) สอน AI ด้วยตัวอย่าง
> การให้คะแนนแค่ 10 นาทีต่อสัปดาห์ สะสมกลายเป็นผู้ช่วยที่ฉลาดขึ้นแบบวัดผลได้จริง

---

## ภาพรวมวงจรเทรน

```
🧪 Prompt Lab                                 📊 Evals
─────────────────────────────────────        ─────────────────────────────
1. เลือก skill + model                        5. กด "Run Eval" เมื่อไหร่ก็ได้
2. วางโค้ด / แนบไฟล์ Z-program        ──►        → AI ตอบข้อสอบ ⭐ ทุกข้อใหม่
3. Run → อ่านคำตอบของ AI                          แล้ว AI-ผู้ตรวจให้คะแนน
4. ตัดสิน (✅/⚠️/❌ + เฉลยที่ถูก)                    เทียบกับเฉลยของ "คุณ"
   → เคสดีๆ: กด ⭐ "เข้าชุดข้อสอบ"              6. คะแนน % บอกว่า prompt/model
                                                 เวอร์ชันใหม่ดีขึ้นจริงไหม
```

## ทีละขั้น: ทดสอบใน Prompt Lab

**Login** ที่ URL ปกติด้วยบัญชี trainer → เข้าหน้า admin → เปิด **Prompt Lab**
ที่แถบเมนูด้านซ้าย

### 1. เลือกสิ่งที่จะทดสอบ
| ตัวเลือก | ทำอะไร |
|---|---|
| **Skill ที่ทดสอบ** | คำถามจะถูกส่งเข้า system prompt ตัวไหน (เช่น *Obsolete Statement Check*) |
| **โมเดล** | AI ตัวไหนเป็นคนตอบ — **Terra** = ใช้ประจำวัน · **Luna** = ถูกสุด (ลองเร็วๆ) · **Sol** = เก่งสุด (เคสยาก) |
| **ระดับการคิด (effort)** | ให้โมเดลคิดนานแค่ไหน — **medium** คือค่าปกติ ปรับเป็น **high** สำหรับคำถาม migration ซับซ้อน |

กด **📄 System prompt ที่ใช้ทดสอบ** เพื่อกางอ่าน prompt ตัวเต็มที่กำลังทดสอบ —
คำถามที่คุณพิมพ์จะถูกตอบ*ด้วย prompt ตัวนี้*

### 2. ใส่โจทย์
- พิมพ์หรือวางโค้ด ABAP / คำถาม ลงในช่อง **โจทย์** หรือ
- กด **📎 แนบไฟล์** เลือกไฟล์ Z-program ได้หลายไฟล์ (`.abap`, `.txt`, …
  ไฟล์ละไม่เกิน 1 MB) — เนื้อไฟล์จะต่อเข้าช่องโจทย์พร้อมหัวคั่น
  `* ===== File: ... =====` แก้ไข/ตัดทอนก่อน Run ได้
- ตัวนับตัวอักษรใต้ช่องบอกขนาดโจทย์ (ยิ่งใหญ่ = token ยิ่งเยอะ = ค่าใช้จ่ายเพิ่ม)

### 3. Run
กด **▶ Run** — คำตอบขึ้นช่องขวาพร้อมจำนวน token ที่ใช้
ทุกการ Run เสียค่า token ของ OpenAI จริง (ปกติไม่กี่บาท) แต่**ไม่หักจาก pool
ของโปรเจกต์ลูกค้าเด็ดขาด** — Lab เป็น sandbox แยกต่างหาก

### 4. ตัดสินคำตอบ — ขั้นที่สำคัญที่สุด
ใต้คำตอบจะมีแถบ **คำตัดสิน (senior)**:

| คำตัดสิน | ใช้เมื่อไหร่ |
|---|---|
| ✅ **ถูกต้อง** | ถูกต้องทางเทคนิคและครบถ้วน ไม่มีประเด็นสำคัญตกหล่น |
| ⚠️ **เกือบถูก** | ทิศทางถูกแต่ไม่ครบ/ไม่เป๊ะ — **เขียนส่วนที่ขาดลงช่องเฉลยที่ถูกต้อง** |
| ❌ **ผิด** | ผิด ชี้นำผิดทาง หรือมองไม่เห็นปัญหาจริง — **เขียนคำตอบที่ถูกลงช่องเฉลยที่ถูกต้อง** |

กรอกเพิ่ม:
- **เฉลยที่ถูกต้อง** *(โชว์เมื่อเลือก ⚠️/❌)* — คำตอบแบบที่*ควรจะเป็น*
  เขียนแบบที่อยากให้ junior ได้อ่าน เพราะนี่คือมาตรฐานที่ AI จะถูกตรวจเทียบ
  **ความแม่นยำตรงนี้สำคัญที่สุดในทั้งระบบ**
- **หมวด** *(ไม่บังคับแต่มีค่ามาก)* — tag สั้นๆ เช่น `FI`, `MM`, `SD`,
  `syntax`, `S4-migration` — รายงานจะแยกคะแนนตามหมวด ทำให้เห็นชัดว่า AI
  อ่อนตรงไหน
- **โน้ต** *(ไม่บังคับ)* — อะไรก็ได้ที่ควรจดจำเกี่ยวกับเคสนี้

กด **💾 บันทึกคำตัดสิน** — เสร็จ เคสถูกเก็บเข้าระบบแล้ว

> **เคล็ดลับ — ตรวจเป็นชุด:** ไม่ต้องตัดสินทันทีหลัง Run ทุกครั้ง
> ทุกการ Run ถูกเก็บใน **📋 ประวัติการทดสอบ** ล่างสุดของหน้า
> กรองด้วย *⏳ ยังไม่ตัดสิน* แล้วเปิดตรวจทีละเคสในคราวเดียวได้
> และเปิดเคสที่ตัดสินแล้วมาแก้คำตัดสินย้อนหลังได้เสมอ

### 5. เลื่อนเคสดีๆ เข้าชุดข้อสอบ (⭐)
เปิดเคสที่ตัดสินแล้วในประวัติ → กด **☆ เข้าชุดข้อสอบ**

- เลือกเฉพาะเคสที่**คุ้มค่าจะสอบซ้ำไปตลอด**: โค้ดจากงานจริง เฉลยชัดเจน
  เป็นตัวแทนของงานประจำวัน
- การเลื่อนขั้นต้องมีเฉลยอ้างอิง: ตัดสิน **✅ ถูกต้อง** (คำตอบ AI เป็นเฉลยเอง)
  หรือมี**เฉลยที่ถูกต้อง**ที่คุณเขียนไว้
- ข้ามเคสซ้ำและเคสลองเล่น — **คุณภาพข้อสอบสำคัญกว่าจำนวน**

**เป้าหมาย: ~30 เคสต่อ skill** กระจายหลายหมวด — ถึงจุดนั้นคะแนนจะเริ่ม
เชื่อถือได้ทางสถิติ

## การสอบ: หน้า Evals

เปิด **Evals** ที่แถบเมนูด้านซ้าย

1. เลือก **skill** และ **model/effort** ที่จะสอบ — หัวการ์ดบอกว่ามีข้อสอบ ⭐
   พร้อมกี่ข้อ
2. กด **▶ Run Eval** — มี progress bar ตามความคืบหน้า (ราว 5–20 วินาทีต่อข้อ)
   สอบได้ทีละรอบ และกดหยุดกลางทางได้
3. อ่านรายงาน:
   - **คะแนนล่าสุด** — % ของเคสที่ผ่าน พร้อมลูกศร ⬆/⬇ เทียบรอบก่อน และ
     เส้นแนวโน้มข้ามหลายรอบ
   - **แถบแยกหมวด** — อัตราผ่านต่อหมวด (tag ที่คุณใส่ตอนตัดสินมีผลตรงนี้:
     *"แข็ง syntax แต่อ่อน SD pricing"*)
   - **รายการเคส** — ทุกเคสพร้อมเหตุผลสั้นๆ ของผู้ตรวจ กดที่เคสเพื่อดู
     *โจทย์ / เฉลย / คำตอบรอบนี้* เทียบกัน

**สอบเมื่อไหร่:** หลังแก้ skill prompt, ตอนลอง model/effort ใหม่, หรือเป็นระยะ
เมื่อชุดข้อสอบโตขึ้น — สอบ 30 ข้อใช้ค่า token ราว ฿30–60 (ไม่แตะ pool ลูกค้า)

**อ่านคะแนนตกยังไง:** ถ้าคะแนนตกหลังแก้ prompt → เปิดเคสที่ตก อ่านเหตุผล
ผู้ตรวจ แก้ prompt แล้วสอบใหม่ — **ห้าม ship การแก้ prompt ที่ทำคะแนนตก**

## กติกาพื้นฐาน

1. **ทุกการทดสอบต้องจบด้วยคำตัดสิน** — คำตอบที่ไม่ถูกตัดสินไม่สอนอะไรเราเลย
   ถ้า Run แล้วต้องให้คะแนน (ทันทีหรือย้อนหลังจากประวัติก็ได้)
2. **เฉลยที่ถูกต้องคือทองคำ** — เขียนให้ครบและแม่น เพราะมันคือมาตรฐานที่
   ทุกอย่างถูกวัดเทียบ
3. **อย่าโกงข้อสอบ** — ตัดสินจากสิ่งที่ AI ตอบจริง ไม่ใช่สิ่งที่อยากให้ตอบ
   60% ที่ซื่อสัตย์เราแก้ได้ แต่ 95% ที่ปลอบใจเราแก้ไม่ได้
4. **โค้ดจริงดีกว่าตัวอย่างที่แต่งขึ้น** — Z-program จากโปรเจกต์จริง
   (โดยเฉพาะตัวเข้าคิว ECC6 → S/4HANA) คือเคสที่ดีที่สุด

## คำถามที่พบบ่อย

**ทดสอบแล้วบริษัทเสียเงินไหม?** แต่ละ Run เสียค่า token ของ OpenAI
(ปกติ < ฿2 ด้วย Terra) — ไม่หักจาก pool ของโปรเจกต์ลูกค้าเด็ดขาด

**แก้คำตัดสินย้อนหลังได้ไหม?** ได้ — เปิดเคสในประวัติแล้วบันทึกคำตัดสินใหม่
ระบบยึดคำตัดสินล่าสุด

**แก้ skill prompt ได้ไหม?** ได้ — หน้า **Skill Prompts** (เฉพาะ trainer)
แก้เสร็จให้ Run Eval ยืนยันว่าคะแนนไม่ตก

**ใครเห็น/ทำอะไรพวกนี้ได้บ้าง?** เฉพาะบัญชี **trainer** เท่านั้น —
บัญชี admin ดูแลผู้ใช้กับการเงิน และแตะข้อมูลเทรนไม่ได้เลย
